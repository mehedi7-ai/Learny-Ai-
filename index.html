Send Text, Image & Video Using Facebook Graph API (v23)

This guide shows simple cURL commands to send text, image, and video messages using Facebook Graph API v23.

🚀 cURL Command Templates

✅ 1) Send Text Message
curl -X POST "https://graph.facebook.com/v23.0/<PAGE_ID>/messages" \
  -H "Authorization: Bearer <PAGE_ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{ "messaging_type": "RESPONSE", "recipient": { "id": "<USER_ID>" }, "message": { "text": "Hello from Graph API v23!" } }'
🖼️ 2) Send Image Message
curl -X POST "https://graph.facebook.com/v23.0/<PAGE_ID>/messages" \
  -H "Authorization: Bearer <PAGE_ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{ "recipient": { "id": "<USER_ID>" }, "message": { "attachment": { "type": "image", "payload": { "url": "<IMAGE_URL>", "is_reusable": true } } } }'
🎥 3) Send Video Message
curl -X POST "https://graph.facebook.com/v23.0/<PAGE_ID>/messages" \
  -H "Authorization: Bearer <PAGE_ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{ "recipient": { "id": "<USER_ID>" }, "message": { "attachment": { "type": "video", "payload": { "url": "<VIDEO_URL>", "is_reusable": true } } } }'

📋 Replace With Your Values

Type your parameters directly into the input fields below to dynamically update the cURL commands in real-time.

Placeholder Meaning Your Custom Value
<PAGE_ID> Your Facebook Page ID
<PAGE_ACCESS_TOKEN> Your Page Access Token
<USER_ID> Recipient PSID
<IMAGE_URL> Direct Image Link
<VIDEO_URL> Direct Video Link
Videos/images must be hosted and publicly accessible.

📌 Notes

  • Page must have pages_messaging permission
  • User must have messaged page first
  • Works for Graph API v23
